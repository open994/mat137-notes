In this video, I will
address a common point of confusion:
what happens when we use quantifiers
with the empty set.
If you haven't watched
the previous video,
"Simple Proofs with Quantifiers,"
I suggest you watch that one first,
because I will be referencing it.
I will put a link in the description.
Have a look at these two statements,
and be careful because there is a trap.
The first one says,
"for every x in the empty set,
x is positive."
The second one says,
"there exists x in the empty set
such that x is positive."
One of them is true;
one of them is false.
I invite you to pause the video;
think about them; decide what you think;
and only after that, continue watching.
I've seen statements like this before:
"for every x in a set, x is positive."
So I'm going
to focus on number 1 first,
and look at what I've learned
in the previous video.
In the previous video, I proved
that for every x in this set A,
x was positive.
And then, I disproved
that for every x in the integers,
x was positive.
How did we prove that it was true
that for every x in a set,
x was positive?
We verified all the elements in the set;
we went through them one by one,
and checked that each one of them
was positive.
How did we prove
that "for every x in a set,
x is positive," that this is false?
We found ONE single element
that was not positive.
So either check all the elements
in the set as positive,
or find a single one that isn't positive.
So let's try to do that
with the EMPTY set,
which has no elements.
I'm going to refer to this part here,
"x > 0," as "*"
and I'll explain why in a moment.
Now, let's see what I have learned.
To prove that
for every element in a set, [something];
to prove that that's true,
I need to verify
that ALL the elements in the set
satisfy that thing.
Notice that it doesn't say
to find one, or many.
You need to verify
that ALL the elements in the set
satisfy it.
So if the set has three elements,
check all three.
If it has only one, check that one.
If it has infinitely many,
check all of them.
If it has zero elements,
then check all zero;
it means don't check anything.
On the other hand,
if I want to prove that it is false
that for every x in a set, [something],
then I need to find ONE element.
No matter how many elements
there are in the set,
I need to find ONE element
that doesn't satisfy the condition.
So think of it from this perspective:
which one of these two things can I do?
And I hope it's now clear
that I CAN prove that it's true,
but I CANNOT prove that it's false.
I can prove that it's true,
because I can check ALL the elements
in the empty set
-- there is nothing to check,
and ALL of them satisfy that.
Since there is nothing to check,
I am done.
But I cannot find ONE element
in the empty set without that condition,
because I cannot find one element
to begin with.
So this is good;
we are able to prove that it's true,
and we are not able to prove
that it's false.
So those two things are compatible. Good.
And therefore,
the first statement is true.
I know some students prefer
to focus only on the second thing and say
"it's true because I cannot prove
that it's false,"
but I think it is useful
to look at both perspectives.
We are ABLE to prove that it's true;
there is nothing to do.
And we are NOT ABLE to prove
that it's false.
And after looking at this,
it should also be clear
that the second statement
is going to be false.
There is not much more to do there.
Now, the moral is,
and I hope you have noticed,
the reason I call this "*"
is that it really didn't matter
what this condition was;
the exact same argument would work.
So in general,
ANY statement that begins
with "for every x in the empty set, 
[whatever],"
that's going to be true.
And any statement that begins
with "there exists x in the empty set
such that [whatever],"
that's going to be false.
